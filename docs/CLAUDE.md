# docs/

SignalCraft Biz documentation index (automation-ready).

## Structure
- `docs/ref/`: reference specs and architecture docs (PRD, schema, etc.)
- `docs/status/`: operational boards and issue tracking
- `docs/daily/`: daily execution logs
- `docs/weekly/`: weekly rollups

## Read Order
1. `docs/status/PROJECT-STATUS.md`
2. `docs/status/OPEN-ISSUES.md`
3. `docs/ref/PRD.md`
4. `docs/ref/schema.md`

## Rules
- Keep status file names stable for automation.
- Add new long-lived specs only under `docs/ref/`.
- Daily session logs go under `docs/daily/YYYY-MM-DD/`.
